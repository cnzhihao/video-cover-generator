# PRD Prompt：多比例视频封面生成器 V1

你是一个资深前端工程师。请实现一个纯前端的视频封面生成器。这个工具不需要后端，不需要数据库，不需要登录系统，不需要 AI API，不需要视频抽帧。

工具目标：

用户选择视频类型，上传一张参考图，填写标题信息后，系统自动生成三种比例的视频封面：

* 16:9
* 4:3
* 3:4

然后支持单独下载三张封面，也支持把三张封面拼成一张长图下载。

这个工具主要用于视频平台上传封面时，在平台内手动裁切不同区域。

---

# 一、产品目标

实现一个浏览器本地运行的视频封面生成器。

用户流程：

1. 选择视频类型：

   * 口播
   * 周报
   * 系列教程

2. 上传参考图。

3. 填写主标题。

4. 填写副标题。

5. 如果选择「周报」，额外填写周标识，例如：

   * `2026-06-W3`
   * `2026-07-W1`

6. 系统实时生成三种比例预览：

   * 16:9
   * 4:3
   * 3:4

7. 用户可以下载：

   * 拼接长图
   * 单独下载 16:9
   * 单独下载 4:3
   * 单独下载 3:4

# 二、视频类型定义

## 1. 口播

参考图通常是竖版人物照片。

视觉目标：

* 类似 Dan Koe 风格的口播封面
* 人物照片作为整体视觉主体
* 背景自然延展
* 黑灰渐变承载文字
* 没有明显分界线
* 标题清楚
* 人物脸部清楚

三比例逻辑：

* 16:9：左侧标题，右侧人物，渐变融合
* 4:3：左侧标题，右侧人物，渐变融合
* 3:4：上方人物，底部标题，底部渐变承载文字

---

## 2. 周报

参考图通常是固定的 Q 版头像。

视觉目标：

* 固定栏目模板
* 每期模板基本一致
* 右下角固定放 Q 版头像
* 用户一眼能知道这是同一个人的周报系列
* 周标识必须明显，例如 `2026-06-W3`
* 周标识放在 `OPC WEEKLY` 下方
* 周标识不能太小，手机上也要看得清
* 周标识不能喧宾夺主，主标题仍然是最大信息
* 主标题和副标题之间不能隔太远，它们应该形成一个紧密的信息组

信息优先级：

1. 主标题
2. 周标识
3. Q 版头像
4. 副标题
5. `OPC WEEKLY` 栏目标识

周报字段：

* 参考图：Q 版头像
* 主标题
* 副标题
* 周标识

---

## 3. 系列教程

参考图通常是电脑截图、网页截图、工具界面截图、操作结果图。

视觉目标：

* 截图是主体
* 不要做成左边大标题、右边小截图的传统分栏
* 截图要足够大，用户能看清是一个真实界面
* 标题作为带衬底的文案块叠加在截图上
* 类似 Dan Koe 风格的带底文字，但主体是电脑截图
* 竖版时允许截图局部放大，不强求完整截图

三比例逻辑：

* 16:9：截图大面积铺开，标题文案块叠加在左上或左下
* 4:3：截图仍然占主体，标题文案块叠加
* 3:4：截图局部放大，标题文案块叠加，不强求完整截图

每种模板都使用 Canvas 渲染。

建议统一函数结构：

```js
function renderCover(type, layout, state) {
  const canvas = document.createElement("canvas")
  canvas.width = layout.width
  canvas.height = layout.height

  const ctx = canvas.getContext("2d")

  if (type === "talking-head") {
    renderTalkingHead(ctx, layout, state)
  }

  if (type === "weekly") {
    renderWeekly(ctx, layout, state)
  }

  if (type === "tutorial") {
    renderTutorial(ctx, layout, state)
  }

  return canvas
}
```

# 十二、周报模板实现规格

## 1. 周报图层结构

```text
Layer 1：深色背景层
Layer 2：微弱装饰渐变层
Layer 3：OPC WEEKLY + 周标识
Layer 4：主标题 + 副标题
Layer 5：右下角 Q 版头像
Layer 6：轻微暗角 / 质感层
```

周报不是照片主导，不要让 Q 版头像占据过大比例。
Q 版头像是右下角品牌锚点，不是主标题的竞争对手。

---

## 2. 周报固定文案

系统自动显示：

```text
OPC WEEKLY
```

用户不需要填写这个字段。

周标识由用户填写，例如：

```text
2026-06-W3
```

周标识放在 `OPC WEEKLY` 正下方。

---

## 3. 周报信息层级

视觉大小关系：

```text
主标题最大
周标识第二明显
副标题第三
OPC WEEKLY 较小但清楚
Q 版头像固定在右下角
```

重点：

* 用户应该先看到整体栏目区域中的 `OPC WEEKLY + 周标识`，然后看到主标题。
* 周标识不能太小，手机缩略图里也要能辨认。
* 周标识不能大到淹没主标题。
* 主标题和副标题之间不要距离太远，它们应该形成同一个信息组。

## 5. 周报背景风格

周报背景建议使用固定深色栏目背景，不要强依赖上传图做背景。

背景可以是：

* 深蓝黑
* 深灰黑
* 轻微径向渐变
* 轻微网格纹理
* 轻微噪点质感
* 右下角头像周围可以有柔和光晕

示例：

```js
const bg = ctx.createLinearGradient(0, 0, layout.width, layout.height)
bg.addColorStop(0, "#0B1020")
bg.addColorStop(0.55, "#111827")
bg.addColorStop(1, "#050505")
ctx.fillStyle = bg
ctx.fillRect(0, 0, layout.width, layout.height)
```

可以添加轻微装饰线条，但不要太复杂。

## 6. 周报文字样式

`OPC WEEKLY`：

```text
字号：较小
字重：700
字母间距：明显
颜色：rgba(255,255,255,0.72)
```

周标识：

```text
字号：比 OPC WEEKLY 大
字重：800
颜色：#FFFFFF
位置：OPC WEEKLY 正下方
手机缩略图可读
```

主标题：

```text
最大
字重：900
颜色：#FFFFFF
最多 2 行
```

副标题：

```text
字号约为主标题 38% ~ 48%
颜色：rgba(255,255,255,0.78)
最多 2 行
和主标题距离不要太远
```

建议比例：

```text
主标题字号：100%
周标识字号：42% ~ 55%
副标题字号：38% ~ 48%
OPC WEEKLY 字号：28% ~ 36%
```

```js
function downloadCanvas(canvas, filename) {
  canvas.toBlob(blob => {
    const url = URL.createObjectURL(blob)
    const a = document.createElement("a")
    a.href = url
    a.download = filename
    a.click()
    URL.revokeObjectURL(url)
  }, "image/png")
}
```

# 十七、拼接长图逻辑

请实现：

```js
function renderCombinedCanvas(canvas16x9, canvas4x3, canvas3x4) {
  const combined = document.createElement("canvas")
  combined.width = 1440
  combined.height = 3330

  const ctx = combined.getContext("2d")

  ctx.fillStyle = "#050505"
  ctx.fillRect(0, 0, combined.width, combined.height)

  // 16:9
  ctx.drawImage(canvas16x9, 0, 0)

  // 4:3
  ctx.drawImage(canvas4x3, 0, 810)

  // 3:4 居中放入 1440 × 1440 区域
  const x = (1440 - 1080) / 2
  const y = 810 + 1080
  ctx.drawImage(canvas3x4, x, y)

  return combined
}
```

不要在拼接长图中加入比例文字、水印、边框。
因为这个长图后续要上传到视频平台，让平台自己裁切。
